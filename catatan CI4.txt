- kalo sebelumnya di CI3 routenya cuman lewat template css, nah disini udah bisa pake file route.php kayak yang di laravel/flask.
- routes juga bisa dikasih parameter fungsi, ga cuman controller/method aja.
- (:any) untuk menangkap inputan user lewat url lalu untuk nangkepnya jadi parameter yang dikirimin pake $1
- (:any) ini inputan apapun
- (:segment) inputan apapun kecuali /
- (:num) inputan angka
- (:alpha) inputan huruf
- (:alphanum) inputan huruf dan angka
- $2 ini untuk nangkep inputan berikutnya
- tapi kalo dibuatin routes yang akses langsung dari controller/parameter maka harus ngebuatin routes satu2 untuk controller/method nya
- penggunaan routes juga bisa di khususin buat salah satu pengguna aja (misal : admin). Contoh kasus : mau bikin controller user yang hanya bisa diakses oleh admin :
	Caranya:
	- bikin folder baru di Controller, buat aja folder Admin
	- bikin file controller Users.php
	- di dalam file kasih tambah namespacenya Admin => App	  \Controller\Admin
	- lalu tambahin use App\Controller\BaseController biar 	  basecontollernya ga error
	- terakhir bikin routes baru di file Routes.php buat kalo 	  akses url /users itu arahinnya ke Admin\Users

- Untuk menghubungkan controller dengan model ada 2 cara, ditulis di modelnya atau di kelas BaseController biar semua controller kebagian model tsb. Caranya bisa diliat di video pak sandhika galih ya!
- Selain itu di controller juga bisa konek ke db tanpa perantara model, caranya bisa liat di kodingan controller penelitian.
